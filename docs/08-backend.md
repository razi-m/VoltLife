# 08 — Backend Structure (Owner: Farhan)

Single FastAPI app. Services are plain Python modules with functions — **no classes-for-the-sake-of-classes, no repositories-of-factories.** Razi hands you one pickle + one function; you orchestrate.

## pipeline.py — the heart (pseudocode)

```python
async def process_job(job_id, battery_ids):
    for i, bid in enumerate(battery_ids):
        feats   = features.from_telemetry(get_summary(bid))
        result  = health_engine.assess(feats)                  # Razi's function
        save_assessment(bid, result)
        site, reasons, impact_b = deployment.decide(bid, result)  # gates → score → greedy
        save_deployment(bid, site, reasons, impact_b)
        aid     = aadhaar.issue(bid)                           # ID + events backfill
        await events.broadcast(assessment_event(bid, result, aid))
        await events.broadcast(deployment_event(bid, site, reasons, impact_b))
        if i % 5 == 0:
            await events.broadcast(impact_event(impact.summary()))
        await asyncio.sleep(PACE_S)        # 0.15 default; env var — tune demo length live
    await events.broadcast(job_done(job_id))
```

Wrap each battery in try/except: log + skip, never kill the job. **One bad row must not end the demo.**

## Build order (maps to doc 10)

1. H1–2: repo, `config/db/orm`, mock server live for Zaki, contracts frozen
2. H2–6: ingestion + seed + `/sites`, `/batteries` list/detail on real DB
3. H6–10: pipeline with **stub assess()** (random-ish but schema-correct) end-to-end, WS broadcasting
4. H10–12: swap in Razi's real `model_v1.pkl` — this is Integration Milestone 2
5. H12–16: deployment engine + impact + aadhaar endpoints
6. H16–20: `/demo/reset`, `/demo/replay`, reject-report polish, `test_smoke.py` green
7. H20+: deploy to Railway, keep-warm ping, then freeze and help Zaki

The stub-assess step is the trick: the full pipeline, WS, and frontend integrate **before** the real model lands, so model swap-in is a 30-minute event, not a 6-hour merge.
